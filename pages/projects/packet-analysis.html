Packet Analysis Lab

A hands-on exercise in network traffic inspection using Wireshark to understand protocols and identify network behavior.

Cybersecurity Lab Completed: 2024

Problem Statement

What Was This Lab About?

The objective of this lab was to analyze a captured network traffic file to understand normal network communications, identify different protocols in use, and detect any unusual or potentially suspicious activities. This exercise helps build foundational skills in network monitoring and traffic analysis, which are essential for cybersecurity professionals.

Setup & Tools

How I Approached This

Tools Used:

  • Wireshark: Open-source network protocol analyzer for capturing and inspecting traffic
  • Sample PCAP file: Pre-captured network traffic for analysis
  • Kali Linux: Linux distribution optimized for security testing

Environment:

I conducted this analysis in a virtual lab environment using VirtualBox to ensure isolation and prevent any impact on production networks. The PCAP file contained simulated network traffic from a small office network.

Step-by-Step Approach

The Analysis Process

  1. File Loading: Opened the PCAP file in Wireshark and reviewed the overall capture statistics
  2. Protocol Identification: Used Wireshark's protocol hierarchy statistics to identify the types and volumes of protocols present
  3. Traffic Filtering: Applied display filters to focus on specific protocols (HTTP, DNS, TCP, UDP)
  4. Packet Inspection: Examined individual packets to understand their structure and payload
  5. Conversation Analysis: Used Wireshark's conversation statistics to identify communication patterns between hosts
  6. Anomaly Detection: Looked for unusual traffic patterns or potential security concerns

Observations & Findings

What I Discovered

Key Findings:

  • Protocol Distribution: HTTP traffic accounted for 45% of packets, followed by DNS (25%) and TCP control packets (20%)
  • Network Topology: Identified 12 unique IP addresses communicating within the network
  • Traffic Patterns: Observed normal web browsing behavior with periodic DNS resolution
  • Security Observations: No encrypted HTTPS traffic detected, indicating potential security concerns

What I Learned

Key Takeaways

This lab reinforced several important concepts in network security and analysis:

  • Protocol Knowledge: Gained practical understanding of how different network protocols appear in captured traffic
  • Analysis Techniques: Learned to use Wireshark's filtering and statistical tools effectively
  • Security Awareness: Recognized the importance of encryption in network communications
  • Ethical Considerations: Understood the legal and ethical boundaries of network traffic analysis

This exercise demonstrated how network monitoring can reveal both normal operations and potential security gaps, emphasizing the need for comprehensive security strategies.
